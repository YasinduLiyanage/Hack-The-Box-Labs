MISSION - using smb, connects to a server using guest/blank password exploitation.

Used tools - nmap, smbclient

1.SMB Protocol

SMB (Server Message Block) is a protocol used to share files, printers and serial ports among Windows machines.

If administrators enable guest or anonymous logins by mistake, it allows access to sensitive data without credentials.

2.Enumeration

I used nmap to see open ports of the target machine and confirmed smb service is running on port 445.

**smbclient -L** command lists the available shares on the server, as you can see above.

There are 4 shares,

->ADMIN$ - this share is about remote admin access

->C$ - this is about system drive shares

->IPC$ - Inter-process communication shares files

->WorkShares - this is a custom human-made share which can have a high chance of misconfiguration.
